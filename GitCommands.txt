Branch Management in GIT
Step 1 :	Create Branch    =====  git branch <"new branch name">
			git branch MyNewBranch
Step 2 :	Checkout Branch
			git checkout MyNewBranch
		Still online it shows one branch so add a new file and then check the git status 
			git status
Step 3 :	git add .
			git commit -m "Updated file GitCommands.txt"
			git push -u origin MyNewBranch
			git checkout master
Step 4 :	Merge new Branch in master branch
			git merge MyNewBranch
			git push -u origin master
			git checkout MyNewBranch
			git status
Step 5 :	To Delete Branch, you must checkout some other branch like master
			git checkout master
			git branch -d MyNewBranch				--To delete from local
			git push origin --delete MyNewBranch	--To delete from remote

Creating tag in git			
		git tag v1.0
		git tag
	Create an annotated tag
		git tag -a v1.1 -m "tag for release ver 1.1"
	to Display the tags	
		git tag
		git show v1.0
		git tag -l "v1.*"
	Push tags to remote
		git push origin v1.0
		git push origin --tags
		git push --tags     				-- To push all tags at once
	Delete Tags localy
		git tag -d v1.0
		git tag --delete v1.1
	Delete tags (If required only) from remote	
		git push origin -d v1.0
		git push origin --delete v1.0
		git push origin :v1.0
What is checksum?
	checksum is a 40character long reference number which is assigned on each git commit. This checksum number is generated by SHA algoritham.
	check the log to see checksum numbers
		git log
	and quit from this command type q and press enter.
Can we checkout the tags?
We cannot checkout tags in git.
We can create a branch from a tag a tag and checkout the branch 
		git checkout -b <branch name> <tag name>
		example: git checkout -b ReleaseVer1 v1.0
Can I create a tag from some past commit?
		Yes, use the command like:
		git tag <tag name> <reference of commit>
	here you don't need the whole 40 character long reference number. You can use part of it(4-5 characters)
	example : git tag v1.2 5fcdb03
